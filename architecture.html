Key Design Patterns

  • Composables over Vuex/Pinia — Shared state via useFilters(), useAuth(), useI18n() singleton composables. Simple, no extra abstraction.
  • Refs + Computed — Raw API data in ref(), derived/filtered values in computed(). Unidirectional data flow.
  • Server-side filtering — All 4 filters sent as URLSearchParams. Backend filters in-memory lists sequentially.
  • Pydantic validation — All API responses typed and validated. Input models for write operations.
  • No persistent storage — JSON files loaded once at startup, all data in memory. Restarts reset state.
  • Custom CSS only — No Tailwind, Bootstrap, or component libraries. Slate/gray design tokens, custom SVG charts.
  • i18n built-in — All UI strings pass through t() from useI18n(). Locale files in client/src/locales/.
